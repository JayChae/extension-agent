## 이게 뭔가

웹으로 하는 반복 업무를, **신입 직원을 가르치듯** 자동화하는 크롬 익스텐션.
가르치면 → 모르면 묻고 → 배워서 기억하고 → 점점 혼자 한다.
(예: 송무팀이 법원 전자소송 사이트 ecfs.scourt.go.kr 업무를 가르쳐 자동화)

---

## 부품 4개

```
 [법원 페이지]
     ▲│  읽기 / 클릭·입력
     │▼
 [Content Script — "손"]   페이지를 읽고 실제로 클릭/입력
     │
 [사이드패널 — 사수 콕핏]   채팅·승인 카드·"Show me"·STOP   ──WebSocket──┐
                                                                      │
 [FastAPI 백엔드 — "두뇌·금고"]  Claude(판단) + 메모리(노트) + 안전잠금 ◄┘
```

| 부품 | 역할 |
|---|---|
| **Content Script** | 화면을 읽어 *번호 붙은 요소 목록*으로 만들고, 실제 클릭/입력 (눈+손) |
| **사이드패널** | 사람이 가르치고·승인하는 채팅창 + STOP 버튼 (닫히지 않는 두뇌 거주지) |
| **백엔드** | Claude(판단) + 메모리(SOP/노트) + 🔒안전잠금 + 🔐비밀 금고. API 키는 여기만 |
| **Claude** | 실제 판단. 스텝 루프=Sonnet 4.6, 가르치기=Opus 4.8 |

## 어떻게 도나 (한 사이클)

```
화면을 본다 → 다음 행동을 정한다 → 실행한다 → 또 본다 → ...
     └ 모르면:  ask_human → 멈추고 사람에게 질문 → 답 들으면 이어감
     └ 위험하면(제출/결제): 무조건 사람 승인  ← 모델이 못 건너뜀
```

- 두뇌는 **단일 에이전트 루프** 하나(`agent.run()`). 우리가 분기 if문·상태머신을 짜지 않음.
- 모델은 요소를 **번호(인덱스)로** 가리킴 — XPath 같은 깨지기 쉬운 주소를 만들지 않음.

## 이 시스템을 특별하게 만드는 4가지

| 기둥 | 한 줄 |
|---|---|
| 🙋 **막히면 묻는다** | 모르면 멈추고 사람에게 질문, 답을 받으면 이어감 |
| 📝 **가르치면 배운다** | "Show me"로 1번 시연/교정 → 모델이 노트(SOP)로 정리 → 사람 승인 → 다음부터 기억 |
| 🎓 **점점 혼자 한다** | 성공률 데이터로 LEARNING→…→AUTONOMOUS 졸업 (단, 위험 행동은 끝까지 사람 승인) |
| 🧩 **필요한 만큼만 읽는다** | 업무를 작은 재사용 스킬로 쪼개 링크로 조립 → 지금 단계만 읽어 토큰 절약 |

---

## 핵심 설계 원칙 3개

1. **코드는 최소, 권한은 모델에게** — 오케스트레이션을 코딩하지 않고 도구만 쥐여줌.
2. **셀렉터 대신 인덱스** — 화면이 바뀌어도 안 깨지게.
3. **위험은 코드로 잠근다** — 제출/결제 같은 비가역 행동은 *신뢰 코드*가 강제로 사람 승인을 받음 (모델 확신도로 판단 안 함).

---

## 기술 스택 (요약)

- **익스텐션:** Chrome MV3 (사이드패널 + 얇은 서비스워커 + content script)
- **백엔드:** Python FastAPI + WebSocket
- **에이전트:** Pydantic AI 단일 Agent (graph 없음) + Claude
- **메모리:** git 버전관리 마크다운 파일 (별도 DB 없음)
